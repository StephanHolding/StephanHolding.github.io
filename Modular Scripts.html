Modular Scripts

Don't invent the wheel twice

What is this

I wanted to create a couple of scripts that I could import into any project. This way I don't have to 'invent the wheel' more than once.

With these scripts modularity is priority number one.

The systems

The following points are systems that I've created. More information about these systems is offered below. The actual scripts can be viewed on another page. To go there, press the "Go to Scripts" button on the right

  1. Serialization
  2. Hierarchy help

Serialization

For serialization I made a data manager that can serialize to XML, for when the file needs to be read, and to binary, for more sensitive files. It can also load these files. There is a base class called SerializableData. Any classes inheriting from this class can be saved by the data manager.

When saving a file, the SaveFile function has to be called. This function has the following paramaters:

  • The file that needs to be saved (inheriting from SerializableData)
  • The path to save the file to (if a non existing path is overloaded, the path will be created)
  • The file name followed by the file extension
  • A Serialization mode. this is an enum in the data manager. here you can choose whether to save to XML or binary.

Hierarchy help

This is a script for simple tasks in Unity's hierarchy.

It contains simple functions like filling a list with a parent and it's children, or changing a parent and it's children's layer. It also has a function for finding every objecting with a certain component. Unity does offer a similar function, but that function does not find inactive gameobjects, while this method does.

This function uses Unity's scene manager to get all the root objects in the game and searches all of them, and their children, for the specified component.
